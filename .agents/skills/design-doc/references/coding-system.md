# 编码体系详细规范

## 文档编码

格式：`L{层级}-{三位序号}` 或 `ADR-{三位序号}` 或 `REF-{三位序号}`，全系统唯一。

示例：`L0-001`、`L2-003`、`ADR-005`、`REF-001`

## 细项编码

格式：`{类型码}-{三位序号}`，全系统唯一，不依赖文档编码。

### 编码生命周期规则

- 序号从 001 开始，在同一类型码内单调递增
- 编码一经分配，**永不删除、永不复用**
- 不再使用的编码将其状态标注为"已废弃"，编码本身保留
- 新增细项始终使用该类型下的下一个可用序号

### 编码与文档的关联

每份文档在末尾维护**细项编码清单**，列出该文档定义的所有细项编码。
`ued/README.md` 维护**全局编码索引**，可按编码查到所属文档。

### 交叉引用规则

- **引用格式**：直接使用细项编码，如 `FR-001`；需要时附标题：`FR-001 用户登录功能`
- **引用方向**：下层引用上层（合法），上层不引用下层（上层全覆盖下层）
- **REF 引用**：任何层级均可引用 REF 编码（外部参考不受层级方向约束）
- **文档引用**：使用文档编码，如 `L2-001`；需要时附标题：`L2-001 用户权限需求`
