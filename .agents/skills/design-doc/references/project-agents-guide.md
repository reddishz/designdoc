# 用户项目 AGENTS.md 生成指南

## 概述

当用户使用 design-doc 技能创建和维护产品设计文档时，可以为其项目生成适合的 AGENTS.md 统一入口。

## 生成场景

### 项目初始化
- 首次使用 design-doc 技能时
- 项目结构发生变化时
- 团队成员需要统一导航时

### 生成时机
- 用户明确要求生成项目 AGENTS.md
- 技能检测到项目缺少统一入口
- 项目文档体系建立完成后

## 生成内容模板

基于用户项目的实际情况，生成包含以下内容的 AGENTS.md：

```markdown
# AGENTS.md

## 项目目录
```
{项目根目录}/
├── ued/                    # 产品设计文档目录
├── docs/                   # 项目文档和规范
└── AGENTS.md               # 统一入口
```

## 可用技能

### 🎨 设计文档规范 (design-doc)
**功能**：规范 {项目编码} 项目的设计文档创建、审查和维护  
**使用**：直接说"创建需求文档"、"检查文档规范性"等  
**位置**：`.agents/skills/design-doc/SKILL.md`

### 🔧 项目特定技能
（基于项目实际需求生成）

## 快速开始

**创建设计文档** → 直接说"帮我创建用户登录需求文档"  
**检查文档规范** → 直接说"检查这个架构文档的规范性"  
**分配编码** → 直接说"为这个新功能分配一个编码"

---

*由 design-doc skill 基于项目结构动态生成*
```

## 动态配置

### 项目编码变量
使用 `{项目编码}` 变量动态插入用户的项目编码：
- 从 `ued/.doc-config.json` 获取
- 从环境变量 `DESIGN_DOC_PROJECT_CODE` 获取
- 默认值：空（不启用前缀）

### 目录结构检测
自动检测用户项目的实际目录结构：
- 扫描项目根目录
- 识别 ued/、docs/、assets/、scripts/ 等目录
- 生成符合实际结构的目录树

### 技能配置
根据项目实际包含的技能生成技能列表：
- 检测 `.agents/skills/` 目录
- 列出可用的技能
- 提供使用指导

## 生成原则

### 精简原则
- AGENTS.md 内容要精炼，只包含必要信息
- 避免详细规范，只提供导航和使用指导
- 保持全局通用性，不设特定格式

### 实用原则
- 提供直接可用的使用指导
- 包含项目特定的目录结构
- 便于团队成员快速上手

### 扩展原则
- 为其他技能预留位置
- 保持结构的可扩展性
- 支持项目结构变化时的更新
