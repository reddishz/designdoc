# 废弃规范改进方案

## 问题分析

### 审计结果概览

根据某项目实际设计文档废弃审计（24+ 个废弃细项），总体合规度仅 **55%**：

| 审核维度 | 合规率 | 状态 |
|---------|--------|------|
| 全局索引同步 | 95% | ✅ 较完善 |
| 文档清单同步 | 95% | ✅ 较完善 |
| 废弃原因说明 | 70% | ⚠️ 有内容但格式不规范 |
| 替代方案标注 | 40% | ❌ 多数缺失 |
| 标题删除线 ~~已废弃~~ | 30% | ❌ 主项目大多用粗体括号代替 |
| 废弃时间 | 20% | ❌ 仅 Gridtrade 细项齐全 |
| 废弃人 | 0% | ❌ 全部 24+ 细项无一有此字段 |

### 根本原因分析

#### 1. 规范约束强度不足

**SKILL.md 中的问题**：
- 废弃细项处理（第 186-200 行）使用的是"处理原则"，缺乏强制性表述
- 与"编码含义锁定"等核心约束的强度不匹配
- 没有将废弃处理纳入"AI 操作约束（MUST COMPLY）"部分

**影响**：AI 和用户可能认为废弃标记是可选的，而非必须的。

#### 2. 字段要求与实际操作脱节

**deprecation-guide.md 中的问题**：
- 要求 5 个字段：细项状态、废弃时间、废弃原因、替代方案、废弃人
- 废弃人字段在 AI 场景下难以自动获取（需从配置或环境变量获取）
- 字段优先级不明确：哪些是必须的，哪些是可选的

**影响**：实际操作中容易跳过"废弃人"等字段，导致合规率低。

#### 3. 废弃流程过于复杂

**deprecation-guide.md 中的问题**：
- 定义了 5 步废弃流程（标记细项、反向追溯、处理引用、评估引用方、更新索引）
- 每步都有详细要求，实际操作中难以严格执行
- 缺少"最小可行废弃"的概念

**影响**：流程过于复杂导致执行率低。

#### 4. 审核规范不够全面

**review-guidelines.md 中的问题**：
- 废弃相关审核项（第 61-65 行）只检查：
  - 废弃流程合规
  - 废弃信息完整
  - 引用更新完整
  - 索引同步更新
  - 变更记录完整
- **缺少关键检查项**：
  - 标题删除线格式 ~~已废弃~~
  - 废弃时间格式
  - 废弃原因格式规范
  - 替代方案标注（如有）
  - 废弃人字段（如可获取）

**影响**：审核工具无法发现格式不规范的问题。

#### 5. AI 字段获取问题

**实际场景**：
- AI 生成废弃标记时，"废弃人"字段难以自动填充
- 如果 `ued/.doc-config.json` 中配置了 `author`，可以使用
- 但废弃操作可能是不同用户进行的，与创建文档的用户不一致

**影响**：废弃人字段合规率为 0%。

## 改进方案

### 方案一：分层标记规范（推荐）

#### 核心思路
将废弃字段分为**必须字段**和**可选字段**，降低最低准入门槛，同时提供可选增强字段。

#### 必须字段（MUST）

```markdown
### {项目码}-{类型码}-{序号} 细项标题 ~~已废弃~~

**细项状态**：已废弃
**废弃时间**：YYYY-MM-DD
**废弃原因**：{具体原因}
```

**说明**：
- **细项状态**：明确标注废弃状态，便于机器识别
- **废弃时间**：追溯废弃时间，便于版本管理
- **废弃原因**：说明废弃原因，便于决策追溯
- **标题删除线**：视觉上明确标识废弃状态

**来源**：
- 细项状态：AI 自动生成
- 废弃时间：AI 使用当前日期
- 废弃原因：从用户输入或上下文提取

#### 可选字段（SHOULD/WHEN AVAILABLE）

```markdown
**替代方案**：{替代细项编码}（如有）
**废弃人**：{废弃人姓名}（可获取时）
```

**说明**：
- **替代方案**：强烈推荐提供，便于追溯关系
- **废弃人**：仅在可获取时填写（从 `ued/.doc-config.json` 获取作者作为默认值）

#### 索引字段优化

全局索引和文档清单中的废弃标记可以简化：

```markdown
| 编码 | 类型 | 标题 | 状态 | 废弃时间 | 备注 |
|------|------|------|------|----------|------|
| CRM-RUL-004 | RUL | 账号格式规范 | 已废弃 | 2024-03-23 | 替代：CRM-RUL-018 |
```

**说明**：
- 状态列标注为"已废弃"
- 废弃时间单独一列
- 备注中简要说明替代方案（如有）

### 方案二：简化废弃流程

#### 核心思路
提供"完整废弃流程"和"快速废弃流程"两种选择。

#### 快速废弃流程（适用于简单场景）

当废弃的细项没有引用或引用很少时，可采用快速流程：

1. **标记细项废弃**（必须字段）
2. **搜索引用**（简单搜索，不强制完全追溯）
3. **更新索引**（必须）
4. **人工确认**（可选）

#### 完整废弃流程（适用于复杂场景）

当废弃的细项有大量引用或影响范围广时，采用完整流程：

1. **标记细项废弃**（必须字段 + 可选字段）
2. **反向追溯引用**（全面搜索）
3. **处理引用关系**（逐个标注或更新）
4. **评估引用方状态**（判断是否级联废弃）
5. **更新索引**（完整更新）

### 方案三：强化 AI 约束

#### 在 SKILL.md 中增加废弃操作的强制性约束

**位置**：`SKILL.md` 第 120 行的"AI 操作约束（MUST COMPLY）"部分

**新增内容**：

```markdown
- **废弃标记规范（MUST COMPLY）**：
  - MUST：废弃细项时必须标记标题删除线 `~~已废弃~~`
  - MUST：废弃细项必须包含细项状态、废弃时间、废弃原因三个必须字段
  - SHOULD：尽可能提供替代方案字段
  - WHEN AVAILABLE：废弃人字段仅在可获取时填写（从配置文件获取）
  - MUST：废弃后必须更新全局索引和文档清单
  - MUST：涉及大规模废弃时必须请求用户确认
```

#### 在常见问题中明确废弃标记的强制性

**修改 SKILL.md 第 276-277 行**：

**修改前**：
```
**Q: 废弃的细项如何处理？**
A: 保留细项，标注废弃状态，追溯所有引用并同步标注废弃。
```

**修改后**：
```
**Q: 废弃的细项如何处理？**
A: 废弃细项必须遵循以下规范（强制要求）：
1. 标题必须添加 `~~已废弃~~` 删除线
2. 必须包含三个字段：细项状态、废弃时间、废弃原因
3. 推荐提供替代方案（如有）
4. 废弃人字段仅在可获取时填写
5. 必须更新全局索引和文档清单
6. 必须追溯并处理所有引用关系

详细规范见 [references/deprecation-guide.md](references/deprecation-guide.md)。
```

### 方案四：完善审核规范

#### 在 review-guidelines.md 中增加详细的废弃审核项

**位置**：`review-guidelines.md` 第 61-65 行之后

**新增内容**：

```markdown
- **废弃标题标记**：废弃细项标题包含 `~~已废弃~~` 删除线
- **废弃必须字段**：包含细项状态、废弃时间、废弃原因三个必须字段
- **废弃可选字段**：替代方案字段（如有），废弃人字段（可获取时）
- **废弃时间格式**：废弃时间格式为 YYYY-MM-DD
- **废弃原因规范**：废弃原因属于标准分类（需求变更、设计优化、技术升级等）
- **替代方案标注**：如有替代方案，必须正确标注替代细项编码
- **废弃人字段**：废弃人字段仅在可获取时填写（不强制要求）
```

#### 增加废弃格式规范检查

**新增审核小节**：

```markdown
### 废弃格式规范检查

#### 标题标记规范
- [ ] 废弃细项标题包含 `~~已废弃~~` 删除线（必须）
- [ ] 不使用粗体括号 `(已废弃)` 等其他格式

#### 必须字段检查
- [ ] 废弃细项包含 `**细项状态**：已废弃` 字段
- [ ] 废弃细项包含 `**废弃时间**：YYYY-MM-DD` 字段
- [ ] 废弃细项包含 `**废弃原因**：{具体原因}` 字段
- [ ] 废弃时间格式正确（YYYY-MM-DD）
- [ ] 废弃原因清晰明确（至少说明废弃的简要原因）

#### 可选字段检查
- [ ] 如有替代方案，正确标注 `**替代方案**：{编码}`
- [ ] 如可获取废弃人，正确标注 `**废弃人**：{姓名}`
- [ ] 不强制要求废弃人字段

#### 索引更新检查
- [ ] 全局索引中该细项状态更新为"已废弃"
- [ ] 文档清单中该细项状态更新为"已废弃"
- [ ] 索引中包含废弃时间信息
- [ ] 索引备注中说明替代方案（如有）
```

### 最小可行废弃标记

在快速废弃场景下，只需满足以下最低要求：

```markdown
### {项目码}-{类型码}-{序号} 细项标题 ~~已废弃~~

## 总结

通过分层标记规范、简化废弃流程、强化 AI 约束、完善审核规范和优化文档说明，可以显著提升废弃规范的遵循度。

**核心改进点**：
1. 明确字段优先级，降低最低门槛
2. 强化 AI 约束，确保自动化执行
3. 完善审核规范，提升审核有效性
4. 简化流程说明，提高可操作性

**关键成功因素**：
- 字段优先级明确（必须、可选、条件）
- AI 约束强度匹配实际需求
- 审核规范覆盖所有关键点
- 实际操作与规范要求一致
