# 项目编码注册表（高级资产 - 默认忽略）

> **⚠️ 严正警告**：本文件为**高级扩展特性**，在 99% 的场景下 **MUST NOT** 被创建或使用。
> 只有当您在同一个 `ued/` 目录下管理多个完全独立的、来自不同产品线的项目时，才考虑启用该文件。
> **默认行为**：直接在 `ued/` 下使用简洁编码（如 `FR-001`）。

## 适用场景

1. **多项目并存**：同一仓库内管理多个独立子项目。
2. **企业级合规**：公司明确要求所有文档必须包含全局统一的项目标识。
3. **跨库引用**：需要频繁引用其他仓库的设计细项。

## 项目编码分配原则

1. **全局唯一**：每个项目编码在企业全局范围内唯一。
2. **语义化**：编码应体现项目或产品线特征（如 `CRM`, `ERP`）。
3. **标准化**：遵循 2-4 位大写字母规范（A-Z）。
4. **获取优先级**：按 `ued/.doc-config.json` > 环境变量 > `DEFAULT` 顺序获取。

## 注册与启用流程

1. **确定编码**：根据 [coding-system.md](../references/coding-system.md) 的建议选择编码。
2. **登记注册**：在本表中添加对应条目。
3. **配置启用**：在 `ued/.doc-config.json` 中设置 `"project_code": "YOUR_CODE"`。
4. **通知 AI**：明确告知 AI “当前项目启用项目编码模式，请在生成 ID 时带上前缀”。
