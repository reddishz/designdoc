# 应用注册表片段（用于多应用顶层 README）

> **⚠️ 说明**：本内容不再作为独立配置文件使用，而是作为多应用模式下 `ued/README.md` 中“应用注册表”章节的参考片段。

## 适用场景

1. **多应用并存**：同一仓库内管理多个独立应用。
2. **企业级合规**：公司明确要求所有文档必须包含全局统一的项目标识。
3. **跨库引用**：需要频繁引用其他仓库的设计细项。

## 项目编码分配原则

1. **全局唯一**：每个项目编码在企业全局范围内唯一。
2. **语义化**：编码应体现项目或产品线特征（如 `CRM`, `ERP`）。
3. **标准化**：遵循 2-4 位大写字母规范（A-Z）。
4. **获取优先级**：按应用级 `README.md` > 顶层应用注册表 > 默认空值顺序获取。

## 应用注册表示例

| app_key | 应用名称 | 目录 | 项目编码 | 负责团队 | 状态 | 备注 |
|---------|----------|------|----------|----------|------|------|
| crm | 客户关系管理系统 | `crm/` | CRM | 销售部 | 使用中 | |
| erp | 企业资源规划系统 | `erp/` | ERP | 运营部 | 使用中 | |

## 注册与启用流程

1. **确定编码**：根据 [coding-system.md](../references/coding-system.md) 的建议选择编码。
2. **登记注册**：在顶层 `ued/README.md` 的应用注册表中添加对应条目。
3. **应用声明**：在 `ued/{app-name}/README.md` 的项目元信息区块中声明 `project_code`。
4. **通知 AI**：明确告知 AI “当前应用启用项目编码模式，请在生成 ID 时带上前缀”。
