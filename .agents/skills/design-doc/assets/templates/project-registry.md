# 项目编码注册表

> 企业级项目编码统一管理，确保编码的全局唯一性和标准化。

## 项目编码分配原则

1. **全局唯一**：每个项目编码在全局范围内唯一
2. **语义化**：编码应体现项目或产品线特征
3. **标准化**：遵循统一的命名规范
4. **可扩展**：预留编码空间，支持未来扩展

## 编码规范

- **长度**：2-4位大写字母
- **字符集**：仅限 A-Z 字母
- **命名建议**：
  - 产品线项目：使用产品缩写（如 CRM、ERP、CMS）
  - 部门项目：使用部门缩写（如 HR、FIN、IT）
  - 公共组件：使用 COM、LIB、SHARED
  - 研究项目：使用 LAB、RES、POC

## 申请流程

1. **提交申请**：填写《项目编码申请表》
2. **技术审核**：技术部审核编码合理性
3. **分配编码**：从预留范围中分配唯一编码
4. **注册登记**：更新本注册表
5. **通知相关方**：邮件通知项目组和相关部门

## 变更管理

### 编码状态

- **预留**：已预留但未分配
- **使用中**：项目正在使用
- **已废弃**：项目结束，编码废弃
- **保留**：项目暂停，编码保留

### 变更流程

1. **项目结束**：将编码状态改为"已废弃"
2. **项目暂停**：将编码状态改为"保留"
3. **编码回收**：废弃的编码原则上不复用
4. **特殊复用**：特殊情况需要架构委员会批准
