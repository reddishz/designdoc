# ADR - 架构决策记录模板

```markdown
# {项目编码}-ADR-{编号}-{决策标题}

| 属性       | 值                             |
| ---------- | ------------------------------ |
| 文档编号   | {项目编码}-ADR-{编号}          |
| 状态       | 提议/采纳/拒绝/废弃/取代      |
| 作者       | {当前用户.作者}                         |
| 日期       | YYYY-MM-DD                     |
| 关联文档   | {项目编码}-L3-{编号}           |

## 背景

描述促使做出此决策的背景、问题和驱动力。
相关约束：CON-{编号}、NFR-{编号}（引用上层编码）

## 备选方案

### 方案一：{名称}

- 优点：{优点}
- 缺点：{缺点}

### 方案二：{名称}

- 优点：{优点}
- 缺点：{缺点}

## 决策

- **DEC-{编号}**：{最终采纳的方案描述}

## 理由

描述选择该方案的核心理由，以及排除其他方案的原因。

## 后果

- **正面**：{正面影响}
- **负面**：{负面影响}
- **需关注**：{后续需要注意的事项}

---

## 细项编码清单

| 编码       | 类型     | 标题     |
| ---------- | -------- | -------- |
| DEC-{编号} | 决策     | {标题}   |

## 变更记录

| 版本 | 日期       | 作者   | 变更说明 |
| ---- | ---------- | ------ | -------- |
| v1.0 | YYYY-MM-DD | {当前用户.作者} | 初始版本 |
```

## 使用说明

1. 替换 `{编号}`、`{决策标题}` 等占位符
2. 状态值从 `提议/采纳/拒绝/废弃/取代` 中选择
3. 确保所有细项编码全局唯一
4. 关联文档引用对应的 L3 文档编号（完整形式含 `{项目编码}`）
5. 维护细项编码清单和变更记录
