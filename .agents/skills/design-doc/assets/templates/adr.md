# ADR - 架构决策记录模板

```markdown
# ADR-{编号}-{决策标题}

| 属性       | 值                             |
| ---------- | ------------------------------ |
| 文档编号   | ADR-{编号}                     |
| 状态       | 提议/采纳/拒绝/废弃/取代      |
| 作者       | {当前用户.作者}                         |
| 日期       | YYYY-MM-DD                     |
| 关联文档   | L3-{编号}                      |

> **编码形式**：默认使用简洁编码（`{类型码}-{三位序号}`，与正文占位符一致）。
> **项目前缀**：仅在当前作用域 `README.md` 已声明 `project_code` 且请求发起方明确确认启用时，在各类编码前加 `{项目编码}-` 前缀（细则见 `references/coding-system.md`）。

## 背景

描述促使做出此决策的背景、问题和驱动力。
相关约束：CON-{编号}、NFR-{编号}（引用上层编码）

## 备选方案

### 方案一：{名称}

- 优点：{优点}
- 缺点：{缺点}

### 方案二：{名称}

- 优点：{优点}
- 缺点：{缺点}

## 决策

- **DEC-{编号}**：{最终采纳的方案描述}

## 理由

描述选择该方案的核心理由，以及排除其他方案的原因。

## 后果

- **正面**：{正面影响}
- **负面**：{负面影响}
- **需关注**：{后续需要注意的事项}

---

## 本文档细项编码清单（仅列出本文档定义/包含）

> **清单范围**：仅收录**本文档**定义或包含的细项，**不是** `ued/` 下的全局完整清单。
> **分配流程**：分配编码前必须先检索当前作用域 `README.md` 的全局编码索引，确认无冲突后再分配；分配后同步更新该索引。

| 编码       | 类型     | 标题     |
| ---------- | -------- | -------- |
| DEC-{编号} | 决策     | {标题}   |

## 变更记录

| 版本 | 日期       | 作者   | 变更说明 |
| ---- | ---------- | ------ | -------- |
| v1.0 | YYYY-MM-DD | {当前用户.作者} | 初始版本 |
```

## 使用说明

1. 替换 `{编号}`、`{决策标题}` 等占位符
2. 状态值从 `提议/采纳/拒绝/废弃/取代` 中选择
3. **编码形式与项目前缀**：见文首说明
4. **本文档细项编码清单**：仅本文档范围；分配前检索当前作用域 `README.md` 全局索引，分配后同步更新
5. 关联文档引用对应的 L3 文档编号
6. 维护细项编码清单和变更记录
