## 使用说明

1. **简洁编码**：默认情况下，直接使用 `类型码-序号`（如 `FR-001`）。
2. **新增文档**：从对应层级的模板创建，更新文档索引表。
3. **分配编码**：通过全局编码索引确定下一个可用编号，避免冲突。
4. **更新索引**：创建新细项时，**必须**同步更新 `ued/README.md` 全局编码索引与对应设计文档文末的「本文档细项编码清单」（仅该文档范围）。
5. **项目编码（高级）**：若项目庞大或涉及多项目协作，请在 `.doc-config.json` 中配置 `project_code` 并启用前缀。
6. **类型码完整性**：所有明确定义的细项类型**必须**在上面的索引中列出，确保全局可见性。
