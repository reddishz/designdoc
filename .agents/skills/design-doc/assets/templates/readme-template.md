# 项目设计文档 README 模板

> 本模板同时适用于单应用模式的 `ued/README.md`，以及多应用模式下的顶层 `ued/README.md` 与应用级 `ued/{app-name}/README.md`。AI **MUST** 先确定 `doc_mode`，再按对应结构填充。

## 使用规则

- **单应用模式**：业务文档直接位于 `ued/` 下，由 `ued/README.md` 同时承担项目元信息、文档入口、编码计数器与全局编码索引。
- **多应用模式顶层 README**：`ued/README.md` 只承担多应用总入口、应用注册表、共享规则与跨应用导航，**MUST NOT** 承担具体应用的细项级编码索引。
- **多应用模式应用 README**：`ued/{app-name}/README.md` 承担该应用的项目元信息、文档导航、编码计数器与应用级全局编码索引。

## 模板 A：单应用模式 `ued/README.md`

```markdown

## 文档模式

| 字段 | 值 |

| doc_mode | single-app |

| 字段 | 值 |
|------|----|
| project_name | {project_name} |
| project_code | {project_code 或留空} |

| author | {author} |

2. 新增细项时，必须同步更新本文档的编码计数器与全局编码索引。
3. 若识别到需新增多个独立应用，应迁移为多应用模式。
```

## 模板 B：多应用模式顶层 `ued/README.md`

```markdown
# UED 多应用总览

## 文档模式

| 字段 | 值 |
|------|----|
| doc_mode | multi-app |

## 总览元信息

| 字段 | 值 |
|------|----|
| workspace_name | {文档域名称} |
| owner | {author} |
| last_updated | {YYYY-MM-DD} |

## 应用注册表

| app_key | 应用名称 | 目录 | project_code | 状态 | 说明 |
|---------|----------|------|--------------|------|------|
| crm | CRM | `crm/` | CRM | active | {说明} |
| oms | OMS | `oms/` | OMS | active | {说明} |

## 共享文档（可选）

| 目录/文件 | 说明 |
|-----------|------|
| `shared/README.md` | 共享文档入口 |
| `shared/L0-vision/` | 共享战略文档 |

## 跨应用规则

- 每个应用必须维护自己的 `README.md`、编码计数器和应用级全局编码索引。
- 顶层 `ued/README.md` 不维护各应用细项级全量索引。
- 跨应用引用必须使用完整编码。
- 新增应用前，必须先在应用注册表登记 `app_key`、目录名和 `project_code`。
```
