## 使用说明

1. **简洁编码**：默认情况下，直接使用 `类型码-序号`（如 `FR-001`）。
2. **新增文档**：从对应层级的模板创建，更新文档索引表。
3. **分配编码**：使用编码计数器快速获取下一个可用编号。
4. **更新索引**：创建新细项时，同步更新全局编码索引和文档内细项编码清单。
5. **项目编码（高级）**：若项目庞大或涉及多项目协作，请在 `.doc-config.json` 中配置 `project_code` 并启用前缀。
